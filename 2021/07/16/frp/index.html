案例

FRP：是一个专注于内网穿透的高性能的反向代理应用，支持 TCP、UDP、HTTP、HTTPS 等多种协议。可以将内网服务以安全、便捷的方式通过具有公网 IP 节点的中转暴露到公网。
Github地址 https://github.com/fatedier/frp/
完整Ref（虽然并不咋好看） https://gofrp.org/docs/

本来这玩意儿的服务端在阿里云的VPC环境的ECS上跑的好好的，某一天线下运维的同志说硬件产品连不上这玩意儿啦，出错啦，十八万火急，怎么办，，，但是测了一下其实服务端是正常的，最后查出来的原因并没出现在Frp上。
但是经过这件事还是暴露出Frp服务并没加入到我们的zabbix监控当中，普通的监控网页返回状态码=200不太适合，需要能够登录到网页中再反馈是否能够正常登录才是我想要的

Requests

Python Requests 许多要求身份认证的web服务都接受 HTTP Basic Auth。这是最简单的一种身份认证，并且 Requests 对这种认证方式的支持是直接开箱即可用
https://docs.python-requests.org/en/latest/user/authentication/

这里输出 0 or 1 对应上zabbix配置的触发器实现管理员登录网页是否正常
